MRC Crossover Points
v1.2 novel contribution: In v1.1, concurrency did not change locality structure. Now it does. Each client has a private hot key subset — merging client streams into a global arrival sequence expands the working set a shared LRU must cover. The charts below show how miss rate shifts as client count increases.

MRC by Client Count — Locality Degrades as Working Set Expands

Core insight: With 1 client, a 10% cache covers most requests. As client count grows, each client's disjoint hot keys expand the effective working set a global LRU must track. Miss rate triples at 20 clients. This is why partitioned caches can be architecturally superior even at equal total cache size.

Miss Rate at 10% Cache vs Client Count

Plateau at 50 clients is within simulation variance — the effect saturates once disjoint hot subsets cover the available key space. A physically meaningful bound, not a model artifact.
